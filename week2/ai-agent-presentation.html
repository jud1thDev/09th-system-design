AI AGENT

AI Agent란 무엇인가?

AI Agent는 주어진 목표를 달성하기 위해 상황을 관찰하고, 필요한 도구를 사용해 스스로 다음 행동을 판단하고 실행하는 AI 애플리케이션입니다.

EXAMPLES

대표적인 AI Agent 예시 4가지

01. 코딩 에이전트

Claude Code처럼 코드베이스를 읽고, 수정하고, 테스트까지 도와주는 개발 보조 에이전트입니다.

02. 업무 자동화 에이전트

이메일, 캘린더, 문서, 업무 도구를 연결해 반복 업무를 대신 처리합니다.

03. 리서치 에이전트

자료를 찾고 비교한 뒤 핵심 내용을 요약해 보고서나 발표 초안을 만듭니다.

04. 생활 관리 에이전트

일정, 운동, 소비, 할 일을 관리하도록 개인이 직접 만든 맞춤형 에이전트입니다.

REQUIREMENTS

AI Agent 실행에 필요한 것

01. 모델

상황을 해석하고 다음 행동을 선택하는 LLM 또는 AI 모델입니다.

02. 오케스트레이션 런타임

계획, 도구 호출, 결과 확인을 순서대로 반복 실행하는 제어 계층입니다.

03. 도구와 API

검색, 파일 작업, 코드 실행, 데이터 조회처럼 외부 세계와 연결되는 실행 수단입니다.

04. 메모리와 상태 저장

작업 맥락, 중간 결과, 이전 실행 기록을 보관해 다음 판단에 활용합니다.

ISSUE 02

서버 병목

원인

Agent 요청은 모델 호출, 도구 호출, 결과 확인이 반복되기 때문에 일반 API보다 처리 단계가 많습니다.

영향

특정 구간에 요청이 몰리면 큐가 쌓이고 전체 응답 시간이 길어집니다.

ISSUE 03

메모리 관리 부담

원인

Agent는 대화 이력, 작업 상태, 검색 결과, 실행 기록을 계속 저장하고 다시 불러와야 합니다.

영향

트래픽이 늘수록 DB와 Vector DB 부하가 커지고, 컨텍스트 관리가 복잡해집니다.

ISSUE 04

시끄러운 이웃 문제

원인

일부 사용자가 무거운 작업을 많이 실행하면 공유 리소스를 과도하게 점유할 수 있습니다.

영향

같은 서버나 워커를 사용하는 다른 사용자의 요청까지 느려질 수 있습니다.
