## 1️⃣ 주제

### 스터디 소개

**《가상 면접 사례로 배우는 대규모 시스템 설계 기초》 1, 2권** 및 추가 자료조사를 바탕으로 스터디와 프로젝트를 진행합니다.

최근 AI 코딩 도구의 발전으로 코드를 AI에게 맡기는 추세로 변화하면서, 개발자에게는 설계 역량이 더욱 강조되고 있습니다. 이에 저는 시스템 설계 및 패턴에 대한 원론적인 공부를 할 필요성을 느꼈습니다. 또한, 평소 다양한 IT 시스템들이 어떤 아키텍처 아래에서 동작하는지, 해당 아키텍처가 어떤 요구사항 및 목적 하에서 구성되었는지 궁금증을 느껴왔어서 스터디를 만들게 되었습니다.

### 스터디 목표

- **다양한 도메인의 환경과 요구사항을 살펴보고, 각 도메인의 시스템이 어떤 아키텍처로 설계되었는지 학습**
  - 매주 구체적인 도메인 또는 시스템 유형을 정하고, 그 특징에 맞게 시스템 아키텍처를 설계해봅니다. 핵심 요구사항을 바탕으로 기술들을 선택하고, 관련 지식을 학습하는 방식으로 구상했습니다.
  - 유사한 요구사항을 가진 시스템들이 공통적으로 갖는 아키텍처 패턴을 이해하고, 차이가 발생하는 이유도 비교합니다.
    - ex. 이벤트 및 메시징, API 통신, 실시간 전달, DB와 캐시, 모니터링 등의 기술이 각 도메인에서 어떤 맥락으로 선택되는 걸까?
  - 조사, 발표, 실습하는 과정에서 각 기술의 장단점과 trade-off를 체화
- **스터디 완주까지 요구사항 분석, 아키텍처 추론, 기술 선택 이유, 구현 결과가 정리된 산출물을 남기는 것을 목표**

### 📋 Rules

- 매주 각 주차 과제에 대한 **PR**을 올립니다.
- 브랜치명은 `주차-이름` 형식으로 생성합니다.
  - ex) `week2-aerim`
- 주차별 폴더(`weekN`)에 각자의 `md` 파일을 작성하여 제출합니다.
  - ex) `week2/aerim.md`
- PR은 스터디가 끝난 후 리더들이 일괄적으로 머지합니다.
